CRITICAL FINDINGS: 3 ISSUES (MUST FIX BEFORE PRODUCTION)
================================================================================

1. MISSING USER AUTHENTICATION & AUTHORIZATION
   Location: /src/actions/wallet.ts, /src/actions/benefits.ts
   Severity: CRITICAL - SECURITY VULNERABILITY
   Impact: Users can steal/modify other users' data
   
   → Users can add cards to ANY player wallet
   → Users can toggle/claim ANY benefit
   → No userId verification on any mutation
   
   FIX: Add session checks and ownership validation to all server actions

2. INSUFFICIENT CRON ENDPOINT AUTHORIZATION  
   Location: /src/app/api/cron/reset-benefits/route.ts (lines 18-26)
   Severity: CRITICAL - SECURITY & DATA INTEGRITY
   Impact: Attackers can trigger unlimited benefit resets (DoS + data corruption)
   
   → String comparison vulnerable to timing attacks
   → No validation if CRON_SECRET env var exists
   → No rate limiting
   → No failed attempt logging
   
   FIX: Use timing-safe comparison, require env vars, add rate limiting

3. COMPONENT PROP TYPE MISMATCH
   Location: /src/components/PlayerTabsContainer.tsx vs CardTrackerPanel.tsx
   Severity: CRITICAL - RUNTIME ERROR
   Impact: Application crashes when displaying player tabs
   
   → PlayerTabsContainer passes 'card' prop
   → CardTrackerPanel expects 'userCard' prop
   → Undefined prop causes TypeError
   
   FIX: Rename prop to match interface (use 'userCard' consistently)

================================================================================
HIGH PRIORITY ISSUES: 6 ISSUES (SHOULD FIX)
================================================================================

4. DIVERGENT ROI CALCULATION LOGIC
   Locations: Card.tsx, calculations.ts, CardTrackerPanel.tsx
   Impact: Users see different ROI numbers on different pages
   
   → Card.tsx uses incomplete logic (ignores benefit types)
   → calculations.ts is more complete
   → SummaryStats uses yet another variation
   
   FIX: Use single implementation from calculations.ts everywhere

5. MISSING ERROR HANDLING IN CALCULATIONS
   Location: /src/lib/calculations.ts
   Impact: Silent data corruption (NaN/Infinity in UI)
   
   → No validation of benefit.stickerValue (could be negative)
   → No validation of benefit.timesUsed (could be null)
   → No validation of benefit.type enum
   
   FIX: Add input validation and error handling

6. TIMEZONE & DST HANDLING BUGS
   Location: /src/lib/benefitDates.ts (lines 20-37)
   Impact: Benefits expire on wrong dates during DST transitions
   
   → Uses local time setHours() instead of UTC
   → Silent failures during DST transitions
   → Server timezone != user timezone
   
   FIX: Always use UTC (setUTCHours) and work with ISO strings

7. DUPLICATE CALCULATION LOGIC IN SUMMARYSSTATS
   Location: /src/components/SummaryStats.tsx (lines 83-137)
   Impact: Household ROI differs from individual card ROI
   
   → Reimplements getTotalValueExtracted
   → Doesn't use getNetAnnualFee()
   → Ignores StatementCredit vs UsagePerk distinction
   
   FIX: Import and use calculation utilities

8. MISSING VALUE OWNERSHIP CHECK IN toggleBenefit
   Location: /src/actions/benefits.ts (line 40-64)
   Severity: HIGH - AUTHORIZATION
   Impact: Users can toggle benefits they don't own
   
   FIX: Add player ownership verification

9. INCORRECT EXPIRATION COMPARISON LOGIC
   Location: /src/components/Card.tsx (line 76)
   Impact: Off-by-one errors in uncaptured value
   
   → Uses > instead of < for expiration check
   → Should use getUncapturedValue from calculations.ts
   
   FIX: Use centralized calculation function
